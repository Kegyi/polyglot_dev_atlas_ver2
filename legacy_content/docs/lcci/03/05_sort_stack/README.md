# 03.05 Sort Stack

## Description

Sort a stack so that the smallest elements are on top, using one additional stack. This maps to LCCI 3.5.

Typical interview angle:
- Use an insertion-sort style process with an auxiliary stack.
- Move elements back and forth to preserve sorted order.
- Avoid arrays/lists as primary storage unless discussing trade-offs.

Reference page: https://leetcode.doocs.org/en/lcci/3.5/

- cpp.cpp
- go.go
- python.py
- typescript.ts
- scala.scala

## Input

No input file is required.

## Expected output

```text
1 2 3 4
```
